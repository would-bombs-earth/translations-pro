# 极译 — 智能多引擎网页翻译

极译是一款 Chrome 浏览器扩展，支持**微软翻译**和**谷歌翻译**双引擎的网页翻译工具。在页面加载时自动检测外文页面并进行原地 DOM 翻译，同时支持划词弹窗翻译、右键一键切换、SPA 页面适配、跨设备域名同步等高级功能。

> 最低 Chrome 版本：110

---

## 功能特性

- **自动翻译**：打开外文网页时自动检测语言并翻译为简体中文，无需手动操作
- **双引擎切换**：支持微软翻译（免费）和 谷歌翻译（免费），自动故障转移
- **原地 DOM 翻译**：直接替换页面文本节点和属性（title/alt/placeholder/aria-label），不依赖 iframe 或覆盖层
- **划词翻译**：在排除域名、简体中文页面及未被自动翻译过的页面上，选中文本即可弹出翻译浮窗，支持炫酷黑客绿特效、精确中文词性标注和回译
- **右键菜单**：页面任意位置右键 →「极译此页」/「恢复原文」，一键切换翻译状态
- **SPA 适配**：自动检测 React/Vue/Angular/Next.js 等框架，延迟等待水合完成，监听路由变化重新翻译
- **Shadow DOM 穿透**：支持 Web Components 内文本的翻译
- **跨设备同步**：排除域名列表通过 Cloudflare KV 同步（3s 快速轮询 + 5min 休眠），CF KV 为单一数据源
- **配置导入导出**：KV 同步配置支持 JSON 文件导入/导出
- **诊断日志**：内置环形缓冲日志系统，覆盖 Content / Background / API / KV 四个模块，可一键导出

## 安装指南

### 开发者模式加载（推荐）

1. 下载本项目全部文件到本地
2. 打开 Chrome 浏览器，地址栏输入 `chrome://extensions/`
3. 打开右上角「开发者模式」开关
4. 点击「加载已解压的扩展程序」
5. 选择项目根目录 `translations_pro/`，点击确定
6. 扩展图标出现在浏览器工具栏即安装成功

### 打包安装

1. 在 `chrome://extensions/` 页面点击「打包扩展程序」
2. 选择项目根目录，生成 `.crx` 文件
3. 将 `.crx` 文件拖入扩展页面完成安装

## 使用说明

### 1. 基本用法

安装后，扩展默认开启**自动翻译**。访问任意非简体中文网页时，页面会自动翻译为中文。

- 点击工具栏**扩展图标**打开控制面板
- 右键页面任意位置，选择「极译此页」或「恢复原文」
- 翻译中的页面会显示绿色状态标记

### 2. 控制面板

点击工具栏图标打开 340px 宽的深色面板，包含以下模块：

| 模块 | 功能 |
| --- | --- |
| 自动翻译开关 | 开启/关闭页面自动翻译 |
| 翻译此页 / 恢复原文 | 手动触发当前页面的翻译或恢复 |
| 引擎选择 | 点击「微软翻译」或「谷歌翻译」卡片切换引擎，延迟数据实时显示，附带官方品牌图标 |
| 排除域名 | 添加/删除不自动翻译的域名，支持添加当前站点 |
| Cloudflare KV 同步 | 填入 API Token / Account ID / Namespace ID，域名列表跨设备自动同步，Cloudflare 官方图标，带连通性状态指示 |
| KV 配置导入/导出 | KV 配置支持 JSON 文件导入/导出，方便备份和迁移 |
| 导出日志 | 一键下载包含 Content、Background、API、KV 四个模块的诊断日志 |

### 3. 划词翻译

在**排除域名、简体中文页面以及未发生自动翻译的页面**上，选中任意外文文本，会弹出深色翻译浮窗，显示：

- 翻译结果（英文单词直接显示词典释义，短语/句子走轻量翻译通道。结果采用炫酷黑客绿发光字体显示）
- 词性标注（名词、动词、形容词等，附彩色分类标签）
- 回译对照

**交互方式：**

- 按 `Esc` 键关闭
- 点击浮窗外部关闭
- 点击右上角 ✕ 按钮关闭
- 快速连续选词时自动丢弃旧请求，只显示最新结果

**性能优化：**

- 英文单词使用**网易有道词典无 Key 公开接口（JSON API）**，无需申请 Key 即可获取完全免费且极度精准的中文词性和释义
- 短语/句子使用轻量翻译通道（`quickTranslate`），跳过引擎选择、标记解析、分块等开销
- 本地 LRU 缓存（最多 50 条），重复选词秒出结果

### 4. 引擎说明

- **微软翻译**：免费，无配置要求，单次请求字符上限 4500
- **谷歌翻译**：免费，无配置要求，单次请求字符上限提升至 4500（采用最新 batchexecute 接口）

> 扩展默认使用微软翻译，当微软翻译不可用时自动切换到 谷歌翻译。

### 5. 跨设备域名同步（Cloudflare KV）

排除域名列表可通过 Cloudflare Workers KV 跨设备同步：

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 进入 Workers & Pages → KV → 创建命名空间（名称随意，如 `jiyi-domains`）
3. 进入「我的个人资料 → API 令牌」→ 创建令牌，权限选 `Account.Workers KV Storage:Edit` + `:Read`
4. 在扩展面板「排除域名」区域填入：
   - **API Token**：上一步创建的令牌
   - **Account ID**：Dashboard 右侧 API 区域可复制
   - **Namespace ID**：KV 命名空间详情页可复制
5. 填入后自动保存，右上角状态点变绿即连接成功
6. 配置支持 JSON 文件**导入/导出**，方便备份和迁移

> 未配置 KV 时域名列表仅本地存储。

### 6. 排除域名管理

- **手动添加**：在输入框输入域名（如 `github.com`），点击添加
- **添加当前站点**：在目标页面打开面板，点击「添加当前站点」
- **删除域名**：展开域名列表，点击域名右侧的删除按钮
- 排除域名列表通过 Cloudflare KV 跨设备同步

## 技术参数

以下参数在 `content-globals.js` 中定义，可根据需要调整：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `MAX_QUEUE` | 200000 | 最大翻译队列长度 |
| `BATCH_SIZE` | 150 | 单次批处理节点上限 |
| `BATCH_CHARS` | 4000 | 每批最多翻译字符数 |
| `CONCURRENT` | 5 | 并行翻译 worker 数 |
| `FLUSH_MS` | 50 | 队列刷新防抖延迟 |
| `HYDRATION_DELAY_MS` | 3500 | SPA 框架水合等待时间 |
| `INCREMENTAL_THRESHOLD` | 2 | 增量翻译队列触发阈值 |

以下参数在 `background-api.js` 中定义：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `MS_LIMIT` | 4500 | 微软翻译单次请求字符上限 |
| `GOOGLE_LIMIT` | 已废弃 | 统一使用 `MS_LIMIT` (4500) 进行长文本分块处理 |
| `FETCH_CONCURRENT` | 2 | 后台并行 API 调用数 |
| `MIN_FETCH_MS` | 250 | API 调用最小间隔（250ms = 4 QPS，低于 5 QPS 限制留余量） |
| `FETCH_COOLDOWN_MS` | 900 | 触发限流后强制冷却时间 |

## 常见问题

### 翻译没有生效？

1. 检查当前域名是否在排除列表中
2. 检查控制面板的「自动翻译」开关是否开启
3. 页面可能已经是简体中文，扩展会跳过
4. 点击面板中的「翻译此页」手动触发

### 翻译引擎不可用？

1. 检查网络连接是否正常
2. 切换到另一个翻译引擎作为备用
3. 查看面板中的延迟数据，显示「不可用」表示连接失败

### Cloudflare KV 同步不生效？

1. 检查 KV 卡片右上角状态点：绿色=正常，红色=失败
2. 确认 API Token 权限包含 `Workers KV Storage:Edit` 和 `:Read`
3. 确认 Account ID 和 Namespace ID 填写正确
4. 检查 KV 命名空间是否已创建

### 页面翻译后布局错乱？

某些网站的 CSS 依赖原始文本长度，翻译可能导致轻微的布局偏移。这是原地翻译的固有局限。如遇到严重问题，可将该域名加入排除列表。

### 如何导出日志排查问题？

1. 打开扩展控制面板
2. 点击底部的「导出日志」按钮
3. 日志文件将自动下载，包含 Content、Background、API、KV 模块的完整日志

## 项目结构

```text
translations_pro/
├── manifest.json          # 扩展清单（Manifest V3）
├── background.js          # Service Worker：右键菜单、消息路由、状态管理、KV 同步
├── background-api.js      # 翻译引擎核心：微软/Google API、缓存、并发调度、词典查询
├── kv-sync.js             # Cloudflare KV 同步：跨设备排除域名同步（3s 轮询 + 5min 休眠）
├── dom-hijack.js          # MAIN 世界劫持：拦截 DOM 属性读取与底层 Fetch/XHR 数据，实现预翻译
├── content-globals.js     # 共享工具：日志、计数器、DOM 规则、参数配置
├── content-lang.js        # 语言检测：零依赖正则引擎、多语言识别
├── content.js             # 页面翻译：DOM 遍历、批量翻译、划词弹窗、SPA 适配
├── debug.js               # 调试工具：控制台彩色批次日志
├── options.html           # 控制面板 UI（深色主题、引擎卡片带官方品牌图标）
├── options.js             # 控制面板逻辑：引擎切换、域名管理、配置导入/导出
├── icons/                 # 扩展图标（16/32/48/128 PNG）
└── README.md              # 本说明文档
```

### 划词翻译关键函数

| 函数 | 文件 | 说明 |
| --- | --- | --- |
| `setupSelectionTranslate()` | content.js | mouseup 监听、选词检测、消息发送、排除域名检查 |
| `showSelPopup()` | content.js | 深色弹窗创建/更新（浮动 ✕ 按钮、入场动画、词性彩色标签），支持原地刷新避免闪烁 |
| `closeSelPopup()` | content.js | 关闭弹窗并清理事件监听 |

| `quickTranslate()` | background-api.js | 轻量 谷歌翻译，绕过全量流水线，含 429 重试 |

## 更新日志

### v1.2.1

- **Bug 修复**：修复了无头预翻译中消息发送类型不匹配导致控制台报出 `Unchecked runtime.lastError` 错误的问题。
- **稳定性**：在 `background.js` 增加消息处理的全局兜底，避免未知消息卡死端口导致异常。

### v1.2.0

- **无头预翻译 (Headless Pre-translation)**：在 `MAIN` 世界拦截 `Fetch/XHR`，提前提取 JSON 中的潜在文本并异步送翻，实现 SPA 动态渲染时的 0 延迟同步命中缓存体验。
- **混合语种检测优化**：重写 `detectSourceLang` 逻辑。当检测到“繁体+大量英文”混排时自动交由引擎判定或强制 `en`，解决了 Google 引擎在此类场景下“只做繁转简，不翻英文”的顽疾。
- **极致增量渲染**：将 `INCREMENTAL_THRESHOLD` 下调至 2，队列防抖降低至 50ms。弹窗、瀑布流新元素的反应速度全面拉满。
- **水合零等待**：移除了原本由于等待 SPA 框架水合而设置的 `HYDRATION_DELAY_MS` 延迟，依靠底层 API 劫持无缝骗过 React/Vue 的渲染校验。

### v1.1.1

- **性能优化**：使用 `requestAnimationFrame` 批量处理 DOM 更新，彻底解决复杂页面的渲染卡顿问题。
- **功能增强**：支持翻译数据导出功能（通过消息通道），方便进行数据备份与分析。
- **兼容性修复**：修复了在 Twitter 等特定网站上由于劫持 `XMLHttpRequest` 导致的部分静态常量（如 `DONE`, `OPENED`）丢失问题。
- **权限优化**：移除了不必要的 `alarms` 权限，减少资源占用。
