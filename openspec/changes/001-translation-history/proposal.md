# 翻译历史记录

## 背景

当前极译扩展在每次翻译后仅将结果缓存（内存 + IndexedDB L2），但不保留翻译活动的历史记录。用户无法回溯之前翻译过的内容、查看某个词或段落的翻译历史、或者了解扩展在指定时间段内的工作量。

常见用户场景：
- 用户阅读外文文章时翻译了某段话，翻回去想再看一次译文却找不到
- 用户想回顾今天翻译了哪些内容、用了哪些引擎
- 用户发现翻译结果不理想，想对比同一段文本在不同时间或不同引擎下的翻译差异

## 动机

1. **用户可追溯**：让用户能查看过去翻译过的原文和译文，形成个人翻译记忆库
2. **用量透明**：让用户了解扩展的实际工作负载（翻译条数、引擎分布、耗时）
3. **调试辅助**：在用户反馈翻译质量问题时，历史记录可帮助开发者复现问题
4. **数据资产**：长期积累的翻译历史可作为用户的个性化语料库

## 影响范围

| 模块 | 影响程度 | 说明 |
|------|---------|------|
| `background-api.js` | **高** | 新增历史记录存储模块，在翻译完成后记录条目 |
| `background.js` | **中** | 新增消息路由（查询/清空历史） |
| `options.html` | **高** | 新增"翻译历史"面板 UI（含列表、筛选、清空） |
| `options.js` | **高** | 新增历史查询、渲染、交互逻辑 |
| `content.js` | **低** | 划词翻译和单词查询结果需要送回 background 记录 |
| `manifest.json` | **无** | 不需要新增权限 |

## 非目标

- 不实现跨设备历史同步（未来可作为独立变更）
- 不实现历史记录的搜索/全文检索（仅基本列表 + 域名过滤）
- 不存储翻译时的页面 DOM 快照或上下文截图
